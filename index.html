Система контроля версий (VCS – Version Control System) – это система с помощью которой можно фиксировать изменения в файлах, и при необходимости возвращатся к ним.

Git – одна из самых популярных распределенных систем. «Распределенность» заключается в том что у каждого разработчика находится полная копия проекта и все сразу могут работать, а на сервере лежит основной репозиторий.

Команда checkout используется с параметрами коммита, ветки и файла. А также с параметрами и без них.

- git checkout “commit hash” – перемещает указатель HEAD на заданный коммит(и состояние проекта) без привязки к ветке, следующий будет ссылаться на этот.

- git checkout “branch” – перемещает указатель HEAD на конец заданной ветки.

- git checkout “name.file” – сбрасывает заданный файл до последнего коммита, даже если он был в индексе.

- git checkout –b “branch” – создает новую ветку и перемещает указатель HEAD на неё.

- git checkout –b “branch ‘commit hash’ ” – перемещает указатель HEAD на заданный коммит и создает новую ветку от него.

- git add “file1” “file2” “file3” – добавляет в индекс файлы.

- git add . – добавляет в индекс все файлы текущей директории.

- git commit – создает снимок (commit) заранее добавленных в индекс файлов.

- git commit -m “message” – добавляет сообщение к коммиту.

- git commit –am “message” – добавляет все файлы текущей директории в индекс и осуществляет коммит с сообщением.

- git pull – загружает удаленное содержимое и сразу пытается изменить состояние локального репозитория, чтобы оно соответствовало этому содержимому(выполняет слияние).

- git push “remote” “branch” – публикует ветку в удаленный репозиторий вместе со всеми необходимыми коммитами и внутренними объектами. При конфликте слияния не перезаписывает. Флаг “—force” используется для принудительной перезаписи.

Зачастую для разработки и последующего тестирования дополнительного функционала отводится отдельная ветка.

– git branch “new branch”(без перехода на new branch) или git checkout –b “new branch” (с переходом).

Если необходимо перенести коммиты из одной ветки (new branch) в другую (master), то можно воспользоваться этой командой git merge. (Выполняем ее, находясь на master в данном случае) git merge “new branch”. Если при этом возможно просто передвинуть вперед указатель ветки, то git так и сделает, иначе будет создан merge commit, который будет ссылаться на два родительских.

Node.js — это JavaScript-окружение построенное на движке Chrome V8.

Node выполняет действие на сервере. С помощью Node можно писать полноценные приложения. Node умеет работать с внешними библиотеками, вызывать команды из кода на JavaScript и выполнять роль веб-сервера.

Node событийно – ориентированный асинхронный движок. При одновременном подключении к серверу многих пользователей Node работает асинхронно, то есть ставит приоритеты и распределяет ресурсы оптимально.

Для установки необходимо перейти на официальный сайт https://nodejs.org и на главной странице скачать последнюю стабильную версию для своей ОС.

Менеджер пакетов — набор ПО позволяющего управлять процессом установки, удаления, настройки и обновления различных компонентов программного обеспечения.

Npm идет в комплекте с node

HTML (HyperText Markup Language — «язык гипертекстовой разметки») — стандартизированный язык разметки веб-страниц.

HTML страница должна содержать DOCTYPE и html в которых находится голова(head) и тело страницы(body).

Блочные элементы занимают всю ширину экрана, строчные по контенту. Для строчных не работает margin-top и margin-bottom

Списки бывают нумерованые (ol) и маркированые(ul), элементы списков – “li”.

Табличное отображение информации осуществляется с помощью тэга "table". "tr" – строка таблицы, "td" – ячейка таблицы

Атрибут — используется для определения характеристик html-элемента и помещается внутри открытого тега элемента. Все атрибуты состоят из двух частей — это имя и значение.

В html имеются теги для форматирования текста.

Ссылка-якорь — это обычная ссылка, в адресе которой используется символ #, после которого следует идентификатор элемента. Идентификатор создаётся с помощью атрибута "id" у того тега, к которому надо перейти при щелчке по ссылке.

Ссылки являются основой гипертекстовых документов и позволяют переходить с одной веб-страницы на другую. Атрибут "href" определяет URL (Universal Resource Locator, универсальный указатель ресурса), иными словами, адрес документа, на который следует перейти.

W3c – организация, разрабатывающая и внедряющая технологические стандарты для Всемирной паутины. Валидный код в разы упрощает индексацию страниц сайта и позволяет им быстрее оказаться в выдаче. Если в HTML коде есть грубые ошибки, то роботы могут запутаться и не найти контент на странице.

Стили определяют внешний вид html элементов.

Для добавления стилей на веб-страницу существует несколько способов:

Связанные – через подключение внешнего файла со стилями.
Глобальные – обьявленые в теге "style"
Внутренние – обьявленые через атрибут "style" в теге

Классы применяются для индивидуального элемента веб-страницы или для одного тега. В классе описаны и сгрупированы стили которые и будут применены.

Идентификатор элемента задается через атрибут “id”, и должен быть уникален.

Существует множество селекторов (селектор атрибутов, тегов, контекстные и т.д.) все они созданы для поиска нужного элемента в документе.

Псевдоклассы определяют динамическое состояние элементов, которое изменяется с помощью действий пользователя.

Псевдоэлементы позволяют задать стиль элементов не определённых в дереве элементов документа, а также генерировать содержимое, которого нет в исходном коде текста. Например маркеры для маркированного списка.
